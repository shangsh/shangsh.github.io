title: "# Chrome 147-148 中的 WebGPU 新动态

Chrome 147 和 148 现已发布，其中包含 WebGPU 的重要更新和问题修复。

## Chrome 148 中的变化

### 适用于 Android 的 GPU 选择器现已推出

在 Android 设备上，用户现在可以直接从浏览器设置中选择 WebGPU 使用的 GPU。可通过访问 **chrome://gpu** 并点击 "GPU 列表" 部分中的下拉菜单来实现。

### 支持多边形渐变三角带

WebGPU 现在支持绘制带有渐变填充的三角形带，渐变从顶点颜色插值。用户可以使用 `WGPULinear Sampling` 或 `WGPUNearest Sampling` 进行纹理采样。

```
wgpu::SurfaceConfiguration surfaceConfig = {
    .width = 512,
    .height = 512,
    .usage = wgpu::TextureUsage::RenderAttachment,
    .format = wgpu::TextureFormat::BGRA8Unorm,
    .viewFormats = {wgpu::TextureFormat::BGRA8Unorm},
    .device = device,
};

surface.Configure(surfaceConfig);
```

### WebGPU 使用指南中的新示例

新增了多个代码示例，包括：

- 渲染到纹理
- 深度/模板缓冲区和延迟着色
- 直接使用 Canvas 渲染

### 废弃的实验性 WebGPU 功能

以下实验性功能已从 Chrome 中移除：

- `exponentialResolutionCubic` 采样模式
- `shader:float32` 语言功能
- `shader:uniformAndStorage8bitStrideAndBaseAlignment` 语言功能

## Chrome 147 中的变化

### 延迟渲染教程

新增了完整的延迟渲染实现教程，包括 G 缓冲区渲染、延迟光照和后处理效果。

### 外部纹理改进

`ExternalTexture` 现在支持更多的纹理格式，包括：

- `R8Unorm`
- `RG8Unorm`
- `RGBA8Unorm-srgb`
- `RGB10A2Unorm`

### 性能改进

- 改进了纹理创建的内存效率
- 优化了渲染通道性能
- 减少了命令缓冲区的内存占用

### 问题修复

- 修复了某些 Intel 显卡上的纹理采样问题
- 解决了 WebGPU 与其他图形 API 之间的上下文丢失问题
- 修复了多线程场景下的渲染问题

## 开发者资源

- [WebGPU API 文档](https://developer.mozilla.org/en-US/docs/Web/API/WebGPU_API)
- [WebGPU 最佳实践](https://web.dev/articles/webgpu-best-practices)
- [WebGPU 性能优化指南](https://developer.chrome.com/docs/web-platform/webgpu-performance)

## 已知限制

- 某些较旧的 Android 设备可能不支持新的 GPU 选择器功能
- 在部分设备上，延迟渲染可能会遇到性能问题

如需了解更多信息，请访问 [Chrome WebGPU 文档](https://developer.chrome.com/docs/web-platform/webgpu)。"
